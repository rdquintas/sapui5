Visual Degradations

Overview of degredations that may apply under certain circumstances in the sap.m library

Depending on the combination of device and browser you are using, the following visual degradations concerning the sap.m library may apply under certain circumstances.

StandardListItem - info (property), DisplayListItem - label (property), InputListItem - content (aggregation)

Certain text elements contained in lists cannot be aligned to the right side of a list if the following conditions apply:

  • You are using Internet Explorer Version 9
  • You are displaying the list inside a popup
  • The contentWidth property of the popup is set to 100% or not at all

These degradations affect the info of a StandardListItem, the value of a DisplayListItem and the input of an InputListItem. The following comparison table shows you exactly what the display should look like (left-hand side) and the visual degradation in each case (right-hand side):

sap.m.Text - maxLines (property), sap.m.Text - text (property), sap.m.ObjectListItem - title (property), sap.m.ObjectHeader - title (property)

The visual aid for indicating multiline overflow is an ellipsis at the end of a line. This ellipsis is displayed if the text string exceeds the maximum number of lines displayed on screen. Depending on the line-clamping support offered by your browser, this visual aid may not be displayed at all, meaning the text is simply truncated without any visual indication that it is incomplete. placeholder Property in sap.m.Input and sap.m.TextArea

As there is no W3C specification for how to use the placeholder property, browser handling for this property varies greatly. Some browsers use a native placeholder property, but for browsers that do not support this, SAP implements its own placeholder version.

The following overview outlines which browsers use which version, and which limitations or degradations apply in each case for the sap.m.Input control and sap.m.TextArea control.

Table 4: placeholder Property in sap.m.Input
Browser Situation
Google Chrome Google Chrome supports the native placeholder property and displays the ellipsis correctly, indicating that the placeholder text stretches beyond the field that is currently visible
Internet Explorer (all versions) All IE versions do not use the native placeholder property, instead they use the placeholder version developed by SAP
Mozilla Firefox Mozilla Firefox supports the native placeholder property but does not display the ellipsis, instead it simply truncates the placeholder text string

Whereas sap.m.Input contains just a single line placeholder, sap.m.TextArea is a multiline control, meaning it brings with it different issues to the ones listed above. These issues are listed below.

Table 5: placeholder Property in sap.m.TextArea
Browser Situation
Google Chrome Google Chrome supports the native placeholder property and displays multiple lines, a scrollbar, and the ellipsis correctly
Internet Explorer (all versions) All IE versions do not use the native placeholder property, instead they use the placeholder version developed by SAP. Where multiple lines of placeholder text are expected, IE browsers only display one line with an ellipsis at the end
Mozilla Firefox Mozilla Firefox supports the native placeholder property but does not display a scrollbar, instead it simply truncates the placeholder text string
