Git Guidelines

Describes the content of th commit message and rules and guidelines for using Git when developing content for SAPUI5

Set the core.autocrlf configuration property in Git to false and make sure to use Unix-style linebreaks (LR-only).

The commit message consists of two or three parts, which are separated by empty lines:

  • The first line contains the commit summary

  • The second line contains an optional commit description text; additional additional empty lines may be contained in the description text

  • Data section

For Git, the following rules and guidelines apply:

  • The summary line must be prefixed with either [FIX] or [FEATURE], which should be followed by the control or component that is the main subject of the change.

  • Exception for this rule is the use of the prefix [INTERNAL] for commits or explanations which are not supposed to be part of the release notes because they are not relevant for SAPUI5 users.

  • The data section consists of name-value pairs:

    • Change-Id (mandatory)

    • Further information, such as Bug-URL for Git-reported bugs, CSS for bugs reported internally at SAP, BCS for customer messages reported at SAP, CR-Id for change requests; the CR-Id is mandatory for maintenance codelines

Example

[FIX] sap.m.Popover: scrolling is removed after Popover is rerendered

- this was caused by the special treatment in dealing with rerendering
in Popover.

- Now the normal invalidation is used and Popup.js takes care of the
focus/blur event listener in onBefore/AfterRerendering
